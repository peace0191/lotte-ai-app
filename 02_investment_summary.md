# 🏠 부동산 AI 플랫폼 - 투자 설명 자료
## 기술 아키텍처 증명서

> **핵심 메시지**: AI가 아니라, AI가 자동으로 진화하는 시스템을 만들었습니다.

---

## 1️⃣ 기술적 차별점

### 🎯 타 경쟁사와의 비교

| 구분 | 일반 부동산 AI | **우리 플랫폼** |
|------|---------------|----------------|
| **모델 관리** | 수동 업데이트 (분기 1회) | **자동 진화 (주 1회)** |
| **판단 근거** | 블랙박스 | **완전 추적 가능** |
| **모델 교체** | 개발자 수동 배포 | **성과 기반 자동 승격** |
| **실험 관리** | 없음 또는 수기 | **MLflow 기반 체계화** |
| **성능 보장** | 없음 | **자동 모니터링 + 롤백** |

### 💡 핵심 기술 스택

```
데이터 수집 → MLflow 실험 관리 → Model Registry → 자동 배포 → 실시간 모니터링
     ↑                                                              ↓
     └──────────────────── 피드백 루프 (계약 데이터) ────────────────┘
```

**우리만의 강점**:
- ✅ 모든 AI 판단이 **설명 가능** (SHAP, Feature Importance)
- ✅ 성과가 좋은 모델이 **자동으로 Production 승격**
- ✅ 실시간 성능 저하 감지 시 **즉시 롤백**
- ✅ 계약 데이터가 다시 학습 데이터로 → **지속적 개선**

### 💰 투자 대비 수익 구조

**초기 투자** (6개월):
- 개발 인력: 2명 × 6개월 = 약 1억원
- 인프라 (AWS): 월 500만원 × 6개월 = 3천만원
- **총 투자: 약 1.3억원**

**예상 수익** (12개월 기준):
- AI 기반 계약 증가: 월 200건 → 연간 2,400건
- 건당 평균 수수료: 100만원
- **연간 추가 수익: 약 24억원**

**ROI**: 약 1,750% (12개월 기준)

## 5️⃣ 경쟁 우위 지속 가능성

### 🔒 진입 장벽

1. **데이터 축적**
   - 계약 이력 데이터는 시간이 지날수록 가치 증가
   - 신규 진입자는 최소 1년 필요

2. **모델 품질**
   - 우리의 모델은 계약 데이터로 지속 개선
   - 경쟁사는 공개 데이터만으로는 한계

3. **네트워크 효과**
   - 중개사 ↔ 고객 양방향 데이터 누적
   - 더 많은 데이터 → 더 좋은 모델 → 더 많은 사용자

## 부록: 기술 용어 설명

**MLflow**: AI 모델 실험을 자동으로 기록하고 관리하는 시스템. Netflix, Databricks 등에서 사용.

**Model Registry**: AI 모델을 버전별로 관리하고, Production/Staging을 명확히 구분하는 시스템.

**A/B 테스트**: 두 개의 모델을 실제 사용자에게 동시에 노출시켜 어느 것이 더 좋은지 실험하는 방법.

**데이터 드리프트**: 시간이 지나면서 데이터의 패턴이 변하는 현상. 이를 감지하지 못하면 AI 성능 저하.

**SHAP**: AI가 왜 그렇게 판단했는지 설명해주는 기술. "이 매물이 저평가인 이유는 전세가율 70%이고, 지하철역과 가깝기 때문"
