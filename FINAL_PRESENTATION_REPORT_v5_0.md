## 0. 프로젝트 요약 (Abstract) - [수정본]
본 프로젝트는 **국토교통부 실거래 데이터(RTMS)**를 기반으로 저평가 매물을 정밀 분석하고, **AI 챗봇**을 통해 고객 상담·매칭·계약 보조부터 **유튜브 숏츠 및 SNS 홍보 콘텐츠** 자동 생성까지 연결하는 **'부동산 중개 밸류체인 풀-자동화 AI 플랫폼'** 구현을 목표로 한다.

특히, 대한민국 교육의 중심지인 **대치1동(래미안대치팰리스, 대치SK뷰, 대치아이파크 등)**의 독특한 국지적 특성인 **'학군 수요'**와 세계 5위 초고층 랜드마크인 **'롯데월드타워몰 시그니엘'**의 자산 가치를 데이터로 입증한다. 본 프로젝트는 단순한 AI 데모를 넘어, **Fast Campus MLOps(Chapter 7~9)** 학습 내용을 실효성 있게 반영하여, 데이터의 유입부터 **고성능 AI 서빙 및 지속적 성능 관리(Continuous Evaluation & Update)**까지 이어지는 **'운영 가능한 완성형 AI 서비스 파이프라인'**을 구현하였다.

## 1. 문제 정의와 사용 시나리오 (4분)
*   **기존 방식의 한계:** 중개사의 주관적 경험에 의존한 매물 추천, 시세 변화 대응 지연, 홍보 영상 제작의 높은 비용과 시간 소모.
*   **국지적 시장의 특수성:** 대치1동은 학교 배정(대도초, 대치초, 대청중 등)을 위한 특정 단지 거주 성향이 매우 강함. 길 하나 차이로 전월세 가격이 수억 원씩 차이나는 특수성을 단순 규칙으로 설명하기 어려움.
*   **AI의 필요성:** 수만 건의 실거래 데이터에서 '학군 프리미엄'과 '저평가 시그널'을 실시간으로 포착하고, 이를 즉시 홍보 콘텐츠(숏츠)로 전환하기 위해 AI가 필수적임.

## 2. 목표 설정과 성공 기준 (3분)
*   **핵심 목표:** 
    1.  RTMS CSV 데이터 입력 시 10초 이내에 저평가 점수 산출 및 통계 업데이트.
    2.  AI 챗봇이 고객의 선호 학교 정보를 바탕으로 최적의 단지를 매칭.
    3.  원클릭으로 실거래 기반의 유튜브 숏츠 대본 및 영상 소스 자동 생성.
    4.  **서빙 최적화:** 다중 요청 상황에서도 1초 이내의 빠른 응답 속도(Latency)와 서비스 안정성 확보.
*   **성공 지표:** 
    *   정확도: 실거래 중위값 대비 오차 범위 3% 이내 시세 반영.
    *   자동화: 데이터 적재부터 숏츠 대본 생성까지의 작업 시간 90% 단축.

## 3. 전체 시스템 구조 및 기술적 배경 (5분)
*   **MLOps 기반 설계:** Fast Campus Chapter 7의 **Container Native Architecture**를 채택하여 서비스 안정성 확보.
*   **Data Layer:** 국토부 RTMS CSV -> SQLite (market_data.db) 자동 적재 (Pipeline 기법 적용).
*   **Logic Layer:** `stats_svc.py`를 통한 중위값(Median) 및 저평가 점수(Scoring) 연산.
*   **AI Agent Layer:** **NER(개체명 인식)** 기술을 응용한 LLM 기반 매칭 엔진 + Prompt Engineering 기반 숏츠 스크립트 생성.
*   **Presentation Layer:** Streamlit Web App (대시보드, 챗봇, 관리자 페이지) + FFmpeg 영상 합성.

### 📚 학습 기술의 프로젝트 투영 (Chapter 7)
| 학습 개념 | 프로젝트 반영 (실무 적용) |
| :--- | :--- |
| **MLOps 필요성** | 실거래 CSV 갱신 시 자동 재분석 및 통계 업데이트 구조 |
| **NER (개체명 인식)** | 자연어 상담에서 지역, 단지, 평형, 거래유형 개체 추출 |
| **Model Serving** | **Triton Server** 구조를 벤치마킹한 독립 백엔드 서빙 (분석/상담/홍보 분리) |
| **Ensemble Backend** | 'NER -> 데이터조회 -> 점수계산 -> 설명생성'으로 이어지는 통합 추론 파이프라인 |
| **Continuous Eval** | **최신 실거래 데이터 기반의 통계 재계산(Re-calculation)** 및 데이터 드리프트 대응 |
| **Reliability** | 표본 수 부족 시 안내 및 과신 방지 로직 (설명 가능한 AI) |

## 4. 데이터 및 지식 설계 (5분)
*   **데이터 출처:** 국토교통부 실거래가 공개시스템 (아파트/오피스텔 매매·전월세).
*   **타겟 단지:** 대치동(은마, 래팰, SK뷰, 아이파크), 신천동(시그니엘).
*   **전처리 전략:** 중복 거래 데이터 제거, 면적별(평형별) 그룹화, 결측치 보정.
*   **지식 베이스(RAG):** 대치동 학군 배정 규칙 및 단지별 특징을 LLM에 사전 학습/컨텍스트 주입.

## 5. 핵심 로직 설계 (5분)
*   **저평가 점수 산식:** `(현재가 - 실거래 중위값) / IQR(사분위수 범위)` 기반의 표준화 점수 적용.
*   **AI 챗봇 체인:** 사용자의 "애가 초등학교 들어가요"라는 질문에서 '대치초/대도초' 키워드를 추출하여 관련 단지를 우선 추천.
*   **숏츠 템플릿:** 30여 개의 마케팅 심리학 기반 대본 템플릿을 AI가 실거래 데이터와 조합하여 생성.

## 6. 검증과 신뢰성 확보 (4분)
*   **데이터 정합성:** 실제 RTMS 파일과 DB 적재 데이터의 대조 검증 루틴 구축.
*   **이상치 및 드리프트 감지:** 거래 가격이 중위값에서 비정상적으로 벗어난 경우 '특수 거래'로 분류하거나 통계에서 제외하여 왜곡 방지.
*   **신뢰성 지표:** 표본 수(Transaction Count)가 부족한 경우 "분석 신뢰도 낮음" 안내를 통해 AI 결과 과신 방지.
*   **환각(Hallucination) 방지:** 시세 정보는 반드시 실거래 통계 DB(rt_stats)의 값만 사용하도록 강제.

## 7. 구현과 실제 동작 데모 (3분)
1.  **RTMS CSV 적재:** 관리자 페이지에서 버튼 클릭 한 번으로 최신 시세 동기화.
2.  **AI 매칭 챗봇:** 고객 상담을 통한 추천 매물 제시 및 계약 예약 대기.
3.  **숏츠 생성:** 동기화된 데이터를 바탕으로 즉시 제작된 숏츠 대본 확인.

## 8. 한계점과 확장 방향 (2분)
*   **한계:** 실거래가 신고 지연(최대 30일)으로 인한 완전한 실시간성 확보의 어려움.
*   **확장:** 네이버 부동산 매물 크롤링을 결합하여 '실제 호가'와 '실거래가'의 괴리 분석(Gap Analysis) 기능 추가 예정. SNS API 연동을 통한 자동 업로드 구현.

## 9. 팀 구성 및 배운 점 [수정본] (1분)
*   **팀 구성:** 본 프로젝트는 개인 프로젝트로 진행되었으나, 2~4차 프로젝트 과정에서 도움을 주신 백경옥 님(음식식단 관리분석 벤치마킹), 한호성 님(행정 질의응답 챗봇 벤치마킹) 등 동료 수강생분들의 아이디어와 자료를 학습에 활용하였습니다. 5차 프로젝트에서는 저의 개인적인 중개 경력을 살려 실무 친화적인 기획을 중심으로 진행하였습니다.
*   **배운 점:** 비전문가로서 인공지능 플랫폼을 활용하여 실무 코드를 생성하고 앱화하는 과정을 통해, AI가 단순한 도구를 넘어 중개 비즈니스의 전략적 파트너가 될 수 있음을 확신했습니다. 
*   **기술적 성장:** 특히 **Fast Campus MLOps (Chapter 7~9)** 과정을 통해 '모델은 만드는 것보다 유지·관리하는 파생 시스템이 더 중요하다'는 것을 깨달았습니다. 데이터 드리프트(금리·정책 변화)에 대응하기 위해 지속적으로 통계를 재계산하고 검증하는 파이프라인을 구축하며, **'설명 가능한 안심 AI'**에 대한 설계 철학을 정립할 수 있었습니다.
